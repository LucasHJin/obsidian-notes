# APIs explained

**Application Programming Interface**
- A software intermediary (set of rules and tools) that allows two applications to talk to each other
	- **In simple terms:** like a waiter in a restaurant who takes your order (request) and brings you the food (response) from the kitchen
	- Gets a request, processes it, returns data or functionality
	- Provides security because the data is never fully exposed in any one step
	- **Examples:**
		- "Log-in Using XYZ"
		- "Google Maps API"
		- "HealthKit API"
