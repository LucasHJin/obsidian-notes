# REST vs HTTP vs CRUD explained

[[CRUD explained|CRUD]] → 4 things you can do with data
- Create, Read, Update, Delete

[[HTTP explained|HTTP]] → a protocol 
- Defines rules for how data moves between client and server
- Defines methods to send CRUD requests

[[REST explained|REST]] → an architectural style
- Uses HTTP as its base
- Defines how to structure API (i.e. use URLs to represent resources, use HTTP methods properly, make stateless requests)
