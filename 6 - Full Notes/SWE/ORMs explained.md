# ORMs explained

**Object Relational Mappers (ORMs):** provide an alternative to writing raw SQL queries
- Widely used in industry 
- Solve problem of not being able to understand database tables and their relations in codebase
	- **Schema:** database definitions into the codebase (can look at columns quickly)
	- Can interact with a SQL database with code (objects/functions) instead of raw SQL
- **Migrations:** changing databases (i.e. adding columns) as project evolves
	- ORMs → standardize migrations with changelogs (no manual rolling)
